## 1. ЦЕЛЬ

Собрать персональный пайплайн: достаём транскрипты созвонов из **Notion AI Meeting
Notes**, Claude делает **кастомное саммари**, рендерим **HTML на Vercel** в заданном
стиле и шлём краткое саммари + ссылку в **Telegram-бот**. Запуск — по расписанию через
**Claude Routines**, без своего бэкенда. Итоговая упаковка — **плагин Claude Code**
с онбордингом. Сначала строим под одного пользователя, затем оборачиваем в плагин.

---

## 2. КОНТЕКСТ (что уже решено и почему)

- **Источник = Notion AI Meeting Notes.** Notion пишет аудио локально с устройства →
  кросс-платформенно (Zoom/Meet/Teams), но приложение Notion должно быть запущено во
  время звонка. Видно только то, что реально попало в Notion.
- **Саммари — обязательно кастомное, делает Claude из транскрипта.** Встроенное
  саммари Notion пользователь считает плохим и ненастраиваемым → его `<summary>`
  максимум как сырьё, основной вход — `<transcript>`.
- **Без своего бэкенда.** Логика живёт в skill плагина + инструкции Routine, Claude
  как агент сам ходит в Notion/Vercel/Telegram. Причина: минимум поддержки, нет хостинга.
- **Источник — сменный адаптер.** Notion это первая реализация; замена платформы
  (Otter/Fireflies/Meet/Zoom/локально) должна стоить «пары мест», не переписывания.
  Причина: продуктизация на широкую аудиторию.
- **Принцип №1: всё параметризовано** (пути, токены, ссылка на Notion — конфиг, не
  хардкод). Причина: дешёвая упаковка в плагин.
- **Продуктизация = плагин Claude Code** (skill + Notion MCP + userConfig). Routine в
  плагин зашить нельзя → плагин даёт `/<plugin>:setup`, проводящий юзера по созданию
  расписания вручную.
- **Разработка ведётся в VS Code**, открыв ИМЕННО эту папку (подхват `CLAUDE.md` + память).

## 3. ОБЪЁМ РАБОТ (по порядку)

1. **Адаптер источника Notion.** Функция «достань встречи за сегодня»: `notion-fetch`
   страницы «Записи встреч» → дочерние страницы → фильтр по дате в заголовке →
   `notion-fetch` каждой с `include_transcript=true`. Маппинг в единый внутренний
   формат `{ date, title, participants, transcript }`.
2. **Модуль саммаризации.** Claude из `transcript` → структурированный JSON
   (`summary`, `decisions`, `tasks[{owner, task, due}]`, `reminders`). Формат саммари
   гибкий, кастомизируемый.
3. **Рендер HTML** из JSON по заданному стилю (референс — см. §4/§7).
4. **Деплой на Vercel** (предпочтительно через CLI).
5. **Отправка в Telegram** (короткое саммари + ссылка на страницу).
6. **Обвязка в Routine** (инструкция расписания, частота — см. открытые вопросы).
7. **Упаковка в плагин Claude Code:** skill, объявление Notion MCP, `userConfig` под
   секреты, команда `/<plugin>:setup` для создания Routine, онбординг (4 поля, §5).

## 4. ТЕХ-ДЕТАЛИ И ОГРАНИЧЕНИЯ

- **Notion-страница источника** «Записи встреч»:
  `https://app.notion.com/p/384474edfa1e8077961df8377aaf50ec`
  Путь: `Мое пространство → Рабочие папки → Записи встреч`.
  Это **страница, НЕ база данных** → нет дат-фильтра, фильтруем по дате в заголовках
  дочерних страниц. Каждая встреча = дочерняя страница, заголовок = дата/время.
- **Notion-доступ** через MCP-коннектор (`notion-fetch`, `notion-search`), подключён и
  проверен 2026-06-19. Страница должна быть расшарена подключению Claude в Notion.
- `notion-fetch ... include_transcript=true` отдаёт `<summary>` (Notion-овское, сырьё) и
  `<transcript>` (наш основной вход).
- **Внутренний формат встречи:** `{ date, title, participants, transcript }`.
- **Рекомендуемый JSON саммари:** `summary`, `decisions`, `tasks[{owner, task, due}]`,
  `reminders` — один источник и для HTML, и для текста в Telegram (не парсить дважды).
- **Деплой:** Vercel CLI предпочтительнее API (меньше движущихся частей).
- **Секреты:** Telegram token + chat_id, Vercel token — в env/keychain/gitignore-файл,
  НЕ в код и НЕ в git. В плагине — через `userConfig` (sensitive → keychain).
- **Запрет хардкода:** пути, токены, ссылка на Notion — только конфиг/параметры.
- **Ограничение плагина:** Routine не упаковывается → только `setup`-команда-гид.

## 5. ВХОДЫ / ВЫХОДЫ

**Входы (доступы/данные):**
- Notion MCP-коннектор + расшаренная страница «Записи встреч» (есть).
- Telegram bot token + chat_id (НЕ собрано).
- Vercel access token (НЕ собрано).
- Референс стиля HTML / брендбук (НЕ подтверждён; кандидат `../zadachi.html`).

**Онбординг плагина собирает у юзера:**
1. Ссылка на Notion-страницу с записями — **с инструкцией** (как взять ссылку + расшарить коннектору).
2. Telegram bot token (+ chat_id) — без инструкции.
3. Vercel token — без инструкции.
4. HTML-файл/брендбук — референс стиля.

**Выходы:**
- HTML-страница на Vercel (выводы по встрече + задачи: кто/что/дедлайн) в заданном стиле.
- Сообщение в Telegram: краткое саммари + ссылка на страницу.
- (Цель) Плагин Claude Code с онбордингом и `setup`-командой.
