# Call Summary — настройка с нуля

Проведи пользователя по шагам. Показывай по одному, дожидайся подтверждения, прежде
чем идти дальше. Если что-то из `userConfig` уже заполнено при установке — не переспрашивай.

## Шаг 1. Подключить источник записей
Спроси, какой источник: **Notion**, **Zoom**, **Telegram (эконом)** или **все сразу**
(`all` — сводка соберёт встречи из всех; дубли не склеиваются, показываются все версии).

**Notion:**
1. Открой страницу с записями созвонов (Notion AI Meeting Notes).
2. Расшарь её подключению Claude (Share → добавить Claude-коннектор).
3. `/mcp` → **notion** → войди в свой Notion. Ссылку на страницу плагин спросил при
   установке (`notion_page_url`); если нет — вставь сейчас.

**Zoom:**
1. Подключи **Zoom** в директории коннекторов Claude (Settings → Connectors → Zoom) —
   войди в свой Zoom, выдай доступ. Это покрывает записи и транскрипты (скоупы
   `cloud_recording:read:*`, `meeting:read:assets`). Токен в этом случае не нужен.
   - Альтернатива для Claude Code: hosted MCP `https://mcp.zoom.us/mcp/zoom/streamable`
     с `ZOOM_MCP_ACCESS_TOKEN` (Bearer). ⚠️ Хост в офиц. источниках расходится
     (`mcp.zoom.us` vs `mcp-us.zoom.us`) — если один не отвечает, попробуй второй.
2. Условие: тариф **Zoom Pro+** с включёнными «Cloud recording» и «Audio transcription»
   (включить **до** созвонов — иначе транскрипт не создаётся).
3. Локальные записи Zoom не видны — нужны именно **облачные**.

**Telegram (эконом — если нет ни Notion, ни Zoom):**
1. Ничего подключать не нужно. В течение дня **пересылай транскрипты созвонов боту**
   (тому же, что шлёт сводки) обычным текстом — одно сообщение = один звонок.
2. Вечером рутина считывает все сегодняшние сообщения (`getUpdates`) и собирает из них
   сводку как обычно. Условие: у бота-приёмника нет вебхука.

## Шаг 2. Токены (если не ввёл при установке)
- **Telegram:** @BotFather → `/newbot` → скопируй **bot token**. Затем напиши своему
  боту любое сообщение и открой `https://api.telegram.org/bot<TOKEN>/getUpdates` →
  возьми `chat.id` — это **chat_id**.
- **Vercel:** vercel.com → Settings → Tokens → Create Token.

## Шаг 3. Создать Routine (расписание) — вручную, в вебе
Плагин не может создать расписание сам (ограничение платформы), поэтому за руку:
1. Открой **https://claude.ai/code/routines** → **Create routine**.
2. **Name:** `Call Summary — ежедневная сводка`.
3. **Schedule:** каждый день вечером (напр. 21:00 твоего времени).
4. **Plugins:** включи плагин **call-summary** (чтобы в облаке был доступен его скилл).
5. **Connectors:** включи **Notion**.
6. **Instructions (промпт):**
   ```
   Запусти скилл call-summary:pipeline — собери сегодняшние созвоны из Notion,
   опубликуй страницу дня на Vercel и пришли сводку в Telegram. Действуй строго
   по инструкции скилла. Ошибки выводи понятно.
   ```

## Шаг 4. Положить секреты в среду Routine
Облачная рутина не видит твою машину — секреты задаются в её среде:
1. На странице рутины нажми иконку среды (cloud / environment) под промптом.
2. Наведись на среду → шестерёнка ⚙️ → **Environment variables**.
3. Вставь три строки (формат .env):
   ```
   TELEGRAM_BOT_TOKEN=<твой токен>
   TELEGRAM_CHAT_ID=<твой chat_id>
   VERCEL_TOKEN=<твой токен>
   ```
4. Там же открой **Network access / Egress** и разреши два домена (иначе деплой и
   Telegram заблокированы firewall'ом песочницы):
   ```
   api.vercel.com
   api.telegram.org
   ```
5. Сохрани.

## Шаг 5. Проверка
- Быстрый тест прямо сейчас: `/call-summary:pipeline` (соберёт сегодняшний день локально).
- Или на странице рутины нажми **Run now** и проверь: пришла ли ссылка в Telegram и
  открывается ли страница дня.

Готово: каждый вечер рутина сама собирает сводку и шлёт ссылку в Telegram.
